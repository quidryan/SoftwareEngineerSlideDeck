<v-click>

A software engineer is someone who **solves problems** using technology.

</v-click>

<v-click>

# What Does Code Look Like?

Here's a tiny example — this is real code that could run on a website:

```python
def can_watch_movie(user_age, movie_rating):
    if movie_rating == "G":
        return True
    elif movie_rating == "PG" and user_age >= 8:
        return True
    elif movie_rating == "PG-13" and user_age >= 13:
        return True
    else:
        return False
```

<v-click>

<div class="callout mt-4">

Code is just **instructions** written in a language the computer understands. It's like writing a recipe — step by step, telling the computer what to do.

</div>

</v-click>

# Why Does School Matter?

---

# Subjects That Actually Help

<div class="columns mt-4">
<div>

### Directly Useful

<v-clicks>

- **Math** — logic, problem-solving, algebra (yes, really)
- **Science** — thinking like a scientist, testing ideas
- **English / Writing** — we write _a lot_ (emails, docs, code comments)
- **Computer Science** — if your school offers it, take it

</v-clicks>

</div>
<div>

### Surprisingly Useful

<v-clicks>

- **Art** — design matters, user interfaces need to look good
- **Music** — patterns, structure, and creative thinking
- **Foreign Languages** — tech is global, teams span the world
- **P.E.** — seriously, sitting all day is rough. Stay active.

</v-clicks>

</div>
</div>

# Skills From School That I Use Every Day

<v-clicks>

| Skill | Where I Learned It | How I Use It |
|-------|-------------------|--------------|

| Writing clearly | English class | Slack messages, documents, code reviews |
| Working in groups | Group projects | Building software with a team |

# You Don't Have to Be a "Computer Person"

<v-click>

I didn't grow up thinking I'd be a software engineer.

</v-click>

<v-click>

<div class="callout mt-4">

What matters most isn't that you already know how to code. What matters is:

</div>

</v-click>

<v-clicks>

- You're **curious** — you like figuring out how things work
- You're **persistent** — you don't give up when something is hard
- You like **solving puzzles** — that's basically what coding is
- You can **work with other people** — software is a team effort
